08
Install Zabbix Agent on EC2
// enables cpu, memory, disk monitoring
bash — SSH into EC2 first
# SSH into your EC2
ssh -i ~/Downloads/Zabbix-keyPair.pem ubuntu@<your-elastic-ip>

# Download Zabbix repo for Ubuntu 22.04
wget https://repo.zabbix.com/zabbix/6.4/ubuntu/pool/main/z/zabbix-release/zabbix-release_6.4-1+ubuntu22.04_all.deb
sudo dpkg -i zabbix-release_6.4-1+ubuntu22.04_all.deb
sudo apt update

# Install agent
sudo apt install -y zabbix-agent

# Find Docker gateway IP (needed for config)
sudo docker inspect zabbix-server | grep -i gateway
# Note the Gateway IP — usually 172.18.0.1

# Configure agent
sudo nano /etc/zabbix/zabbix_agentd.conf
# Set these values:
Server=127.0.0.1,172.18.0.1
ServerActive=127.0.0.1
Hostname=EC2-Zabbix-Server

# Start and enable agent
sudo systemctl enable zabbix-agent
sudo systemctl start zabbix-agent
sudo systemctl status zabbix-agent
🐳
Docker Network Note Because Zabbix server runs inside Docker, it communicates with the host agent via the Docker bridge gateway IP (typically 172.18.0.1), NOT via 127.0.0.1. Always verify with docker inspect zabbix-server | grep Gateway.

Destroy & Rebuild (Data Safe)
// how to teardown without losing zabbix config
terraform — safe teardown
# Step 1: Make sure prevent_destroy is enabled in main.tf
# lifecycle { prevent_destroy = true }  ← must be uncommented

# Step 2: Save your EBS volume ID
terraform output ebs_volume_id

# Step 3: Destroy only compute (EBS volume stays safe)
terraform destroy \
  -target=module.ec2 \
  -target=aws_volume_attachment.zabbix_data_attach \
  -target=aws_eip.zabbix_eip \
  -target=module.vpc \
  -auto-approve

# Step 4: Rebuild everything — data auto-mounts
terraform apply -auto-approve

# Zabbix starts with ALL previous config intact ✅
🔒
Why Data Survives MySQL data is bind-mounted to /data/zabbix/mysql_data on the EBS volume. The user_data script detects the existing filesystem (skips format), mounts it, and starts Docker. Your hosts, triggers, media types, and all config come back automatically.
